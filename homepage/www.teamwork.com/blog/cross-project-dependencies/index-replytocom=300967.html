NEW – Cross Project Dependencies

Posted August 10, 2015

Not every project is straightforward and often the outcome of one part of a project depends very heavily on another one. For a while now in Teamwork Projects, you’ve been able to set up dependencies within a single project, ensuring that a certain task couldn’t¬†start until another one has been completed.

We’ve just introduced Cross Project Dependencies for Teamwork Projects and they’re pretty awesome. Now you can make a task in one project dependant on the completion of a task in a completely different project. It’s something many of you have asked for and we’re delighted it’s finally here.

You can read more about it in our helpdocs section here¬†and as always your feedback on the new feature would be massively appreciated, either in the comments below or by e-mailing us at feedback@teamwork.com.
